What This Actually Does

Prerequisites

Make sure you have these installed:

  • Node.js v18.0.0 or higher
  • npm v9.0.0 or higher
  • Git v2.30.0 or higher
  • 4GB RAM available
  • Azure CLI (optional, for Azure DevOps features)
# Check your versions
node --version
npm --version
git --version

Installation

The package is ready for testing and future npm publishing:

🚀 Current Status: Local Testing

cd agentic-sdlc-platform && npm link

✅ Currently available via npm link

📦 Future: NPM Registry

npm install -g @policy2control/agentic-sdlc-platform

⏳ Available after npm publish

Available CLI Commands

After installation, use these commands:

# Show all available commands
agentic-sdlc --help

# Initialize platform in your project
agentic-sdlc setup

# Check platform status
agentic-sdlc info

# Validate project compliance
agentic-sdlc validate

# Manage workflows
agentic-sdlc workflow list
Platform Status: Initializes configuration in ~/.agentic-sdlc/

Configure Git Hooks

The main value: automatic workflow enforcement

# In your project directory, configure Git to use the hooks
git config core.hooksPath .githooks

# Verify it's configured
git config core.hooksPath
.githooks

Now every commit will be validated for:

  • Work item ID in branch name
  • Proper branch naming convention
  • File organization compliance
  • Security and quality checks

Verify Everything Works

Test that the hooks are active:

# Try to commit without a work item ID
git commit --allow-empty -m "test"

🔍 CLAUDE.md Enhanced Pre-Commit Enforcement
Branch: main
❌ Direct commits to main/master are not allowed
Please create a feature branch first

If you see enforcement messages, it's working!

Enforcement in Action

The platform enforces these policies automatically:

🌿 Branch naming: feature/[work-item-id]-description
📝 Commit messages: Must follow conventional format
🔗 Work item linking: All work must tie to Azure DevOps
🚫 Protected branches: No direct commits to main
📁 File organization: Repository structure compliance

Azure DevOps Integration

If you use Azure DevOps, the hooks will enforce:

# Example: Creating a proper branch
git checkout -b feature/6154-package-platform

# The hook validates:
✅ Work item #6154 verified
✅ Branch name follows convention
✅ Ready for development
Note: You need Azure CLI installed and configured for full Azure DevOps integration
